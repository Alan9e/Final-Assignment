Accessibility

Inclusive design

We started out discussion on the layout wanting to keep it simple. Keep it simple, keep it stupid. We have a simple menu that convey information in plain simple way. Since have not been asked to include elements like sidebars, or make an online store mockup. Our text is centered on the page like an article.

Screen sizes

We have used CSS @media to control how our page behaves on different screen sizes. Thanks to developer tools in browsers we can emulate an iphone 12 for example, to see how it would look on a small screen.

The margins are also adjusted for for different screen sizes. We did consider font size adjustment on smaller screens, but real life testing showed text to be ok. There is of course a concern for people who are visually impaired, and perhaps a dynamic zoom would be the solution. But we have not been able to make that work properly

Reuse of code

Our navigation bar is reused for all pages. Page layout is the same for all pages except FUN, following the same flex-container and div-class setup.

Colors

After we settled on our nav-bar color, we used an online palette generator to find colors that would go well with the nav-bar. You can see our selection here: https://coolors.co We used the softest color in that palette for our background color, it is relaxing on the eyes and goes well with our black font. The contrast ratio for our background color and text sits at close to 20:1

Code validation

We used online code review of html with achecker.achecks.ca and CSS via jigsaw.w3.org We did elect to ignore one problem, and that was replacing i with em in the navigation bar, as that would break elements of the navigation.

Other interesting potential problems were alt text for our image of lagrangian points, the validator does not seem to like any description there.
